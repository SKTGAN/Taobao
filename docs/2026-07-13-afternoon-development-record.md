# 淘宝辅助购买项目说明与 2026-07-13 下午开发记录

## 1. 文档目的

本文记录 2026 年 7 月 13 日下午对项目进行的主要改造、问题修复、验证结果和当前使用边界，便于后续开发、GitHub 协作和问题排查。

本次工作的阶段目标是：先完成单账号、单商品、单任务的定时辅助购买流程。用户提前在可见的 Google Chrome 中登录淘宝、打开商品并人工核对 SKU、数量、收货地址和价格；用户对当前任务进行一次明确授权后，程序在指定时间尝试进入确认订单页面并提交订单，识别到淘宝待付款或收银台页面后停止，不执行自动付款。

## 2. 项目基本说明

### 2.1 项目定位

这是一个运行于 Windows 的 Python/Flet 桌面管理工具，主要由以下部分组成：

- Flet 管理界面：管理账号备注、商品、任务和运行日志。
- SQLite 本地数据库：保存账号备注、商品信息、任务状态和事件日志。
- 独立 Chrome 用户资料目录：由 Chrome 自己保存扫码登录状态。
- Chrome DevTools Protocol（CDP）：在本机连接可见 Chrome，读取有限页面状态并执行经过限制的点击动作。
- 单账号任务执行器：等待指定时间，执行商品页到确认订单页、再到待付款页的流程。

程序入口为 `main.py`，Windows 推荐使用 `start-v2.ps1` 启动。默认管理页面地址为 `http://127.0.0.1:8550`。

### 2.2 当前阶段支持的流程

1. 添加一个账号备注。
2. 启动该账号专用的 Google Chrome。
3. 用户通过淘宝 App 扫码登录。
4. 添加淘宝或天猫商品链接、数量和 SKU 备注。
5. 创建单账号定时任务，计划时间支持毫秒。
6. 点击“准备商品”，人工选择并核对 SKU、数量、价格和地址。
7. 页面预检通过后，用户对该任务进行单次授权。
8. 到达计划时间后点击“立即购买”或同类操作。
9. 等待并识别确认订单页面，然后点击“提交订单”。
10. 检测到待付款或官方收银台页面后停止。

### 2.3 安全边界

当前版本明确不包含以下能力：

- 不保存或读取淘宝密码。
- 不导出或上传 Cookie、LocalStorage 和浏览器登录资料。
- 不处理或绕过验证码、滑块、短信验证和账号保护页面。
- 不伪造浏览器指纹，不轮换 Cookie、UA、代理或 IP。
- 不自动选择 SKU；SKU、数量、地址和价格由用户人工核对。
- 不自动付款，不保存任何支付密码或支付密钥。
- 不承诺抢购成功率，也不保证不会触发淘宝平台风控。

任务授权只对当前任务有效。程序重启后，未完成任务需要重新准备并重新授权，避免旧授权被意外复用。

## 3. 今天下午完成的主要改造

### 3.1 商品链接标准化与空白页问题修复

新增 `src/product_urls.py`，统一处理淘宝和天猫商品链接：

- 从带有跟踪参数的链接中保留有效商品 ID。
- 避免商品链接中的 `&` 在 Chrome 新建标签页接口中被截断。
- 拒绝无法识别商品 ID 或不属于支持域名的链接。
- 商品保存到数据库前统一转换为规范链接。

该修改解决了点击“打开商品”后出现 `about:blank`，或者只打开 `item.htm?abbucket=...` 而缺少商品 ID 的问题。

### 3.2 本机 Chrome DevTools 通信层

新增 `src/cdp_client.py`，实现轻量级 CDP 客户端：

- 通过 Chrome 提供的 WebSocket 页面控制通道发送命令。
- 支持一次性命令调用和可复用的连续会话。
- 为每个命令分配 ID，并只接收对应命令的响应。
- 将连接、超时和 Chrome 返回错误转换为明确的本地异常。

最后一次修复将鼠标移动、按下和释放改为在同一个 WebSocket 会话中执行，并把真实页面控制超时提高到 8 秒，解决淘宝页面较重时连续重连导致的 3 秒超时。

### 3.3 页面识别与有限动作定位

新增 `src/page_automation.py`，负责页面信息采集和分类：

- 识别商品页、确认订单页、登录页、验证页、待付款页和收银台页面。
- 将临时 `about:blank` 与真实加载失败区分开，避免预检过早失败。
- 检查“立即购买”“提交订单”等操作是否存在且可用。
- 遍历普通 DOM、开放的 Shadow DOM 和同源 iframe。
- 计算目标操作控件在顶层页面中的可点击坐标。
- 对页面正文和操作文本进行长度限制，只用于本地状态判断。

页面识别规则不会读取密码、Cookie、支付信息，也不会尝试处理验证页面。

### 3.4 可见 Chrome 会话复用与新标签页跟踪

扩展 `src/safe_browser.py`：

- 每个账号继续使用独立的 Chrome 用户资料目录。
- Chrome 调试端口仅绑定本机 `127.0.0.1`。
- 应用重启后通过 `DevToolsActivePort` 重新连接仍在运行的账号 Chrome。
- 跟踪商品页打开的确认订单新标签页，以及确认订单页继续打开的收银台或待付款标签页。
- 当 Chrome 没有提供 `openerId` 时，使用打开前的标签页基线识别新页面。
- 页面优先级调整为：待付款/收银台、验证页面、最深层新标签页、确认订单页、商品页。
- 点击动作改为 CDP `Input.dispatchMouseEvent` 的真实浏览器鼠标事件，不再只调用 DOM `element.click()`。
- 页面跳转在鼠标释放后关闭旧页面通道时，不再把已经发送成功的点击误判为失败。

### 3.5 单账号定时任务状态机

新增 `src/task_runner.py`，实现单账号任务执行流程：

- 解析带毫秒的计划时间。
- 使用系统时间确定目标时刻，并使用单调时钟完成最后等待。
- 未授权任务禁止自动执行。
- 到点后依次执行购买动作、确认订单识别、提交订单和待付款识别。
- 支持取消正在等待的任务。
- 登录失效、验证页面、按钮不可用或页面无法识别时转为“需人工处理”。
- 浏览器连接失败、超时或程序异常时记录明确失败原因。
- 日志中的页面地址会移除查询参数，减少订单或用户相关信息泄露风险。

当前主要任务状态包括：

```text
草稿 -> 预检中 -> 待授权 -> 已武装 -> 等待中
     -> 触发中 -> 提交中 -> 待付款
```

异常分支包括“需重新准备”“需人工处理”“失败”和“已取消”。

### 3.6 数据库结构与授权持久化

扩展 `src/v2_store.py`：

- 商品链接在写入前进行标准化。
- 任务保存授权时间、触发时间、完成时间、尝试次数和最后错误。
- 增加任务计划时间更新、授权、清除授权、标记触发和完成等操作。
- 应用启动时撤销未完成任务的旧授权，并改为“需重新准备”。
- 数据库迁移兼容已有本地数据，不要求用户删除旧数据库。

本地数据库位于 `data/taobao_assistant_v2.db`，`data/` 已加入 `.gitignore`，不会被正常 Git 提交。

### 3.7 管理界面改造

扩展 `src/gui_v2.py`：

- 任务中心增加“准备商品”“授权”“停止”和“打开购物车”等操作。
- 准备商品后异步等待页面真正完成加载，而不是在临时空白页上立即判断。
- 授权弹窗要求用户再次确认 SKU、数量、地址和价格，并允许修正计划时间。
- 状态栏展示任务状态及最近失败原因。
- 运行日志记录扫码、登录检查、商品预检、授权、调度、点击和异常。
- 限制同一时间只运行一个单账号任务，避免多个任务同时控制同一浏览器。

## 4. 今天处理过的主要故障

### 4.1 商品页打开后为空白

现象：Chrome 地址为不完整的 `item.taobao.com/item.htm?...`，页面为空白。

原因：商品链接没有规范化，且通过 Chrome 调试接口新建标签页时，URL 中的 `&` 被错误处理。

处理：增加商品 URL 标准化与新标签页 URL 编码。

### 4.2 商品已打开但任务显示失败

现象：商品页面正常，任务仍提示当前页面为 `about:blank` 或不可识别页面。

原因：新标签页创建后，程序在页面导航完成前进行了预检。

处理：忽略过渡期空白页，等待实际商品页或明确异常页面出现。

### 4.3 确认订单页的提交按钮不可用

现象：进入确认订单页后提示提交按钮不可用。

原因：按钮可能延迟渲染，且可能位于 Shadow DOM 或同源 iframe 内。

处理：增加最长 15 秒的按钮等待，并递归检查开放 Shadow DOM 和同源 iframe。

### 4.4 点击提交后仍停留在确认订单页

现象：日志显示页面类型为 `confirm_order`，路径仍为 `buy.taobao.com/auction/buy_now.jhtml`。

原因：简单的 DOM `element.click()` 没有被真实淘宝页面完整接受，或者新页面没有被正确跟踪。

处理：改为 Chrome DevTools 的真实鼠标输入事件，并增加多层新标签页与无 `openerId` 情况下的基线跟踪。

### 4.5 修改鼠标点击后，连确认订单页也无法进入

现象：日志显示“无法向 Chrome 发送鼠标点击”“页面控制通道 timed out in 3.0s”。

原因：鼠标移动、按下和释放每一步都重新建立 WebSocket，真实淘宝页面负载较高时连接超时。

处理：在单一可复用连接中完成定位、移动、按下和释放，将超时提高到 8 秒，并正确处理点击后页面立即跳转造成的旧通道关闭。

## 7. 当前推荐使用步骤

1. 在 PowerShell 中进入项目目录。
2. 执行 `./start-v2.ps1` 启动项目。
3. 在“账号管理”中启动账号 Chrome 并扫码登录。
4. 在“商品管理”中添加并检查规范商品链接。
5. 创建计划时间晚于当前时间的单账号任务。
6. 点击“准备商品”，在淘宝页面人工选择正确 SKU 和数量。
7. 确认价格、地址及协议选项后回到任务中心。
8. 点击“授权”，再次确认并提交单次授权。
9. 到点前保持账号 Chrome 和程序运行。
10. 到点后观察 Chrome 和运行日志；进入待付款后由用户自行决定是否支付。

## 8. GitHub 提交注意事项

提交本版本时，应提交 `src/`、`tests/`、`docs/`、依赖文件、README 和启动脚本等源代码与文档。

禁止提交以下本地文件：

- `.venv/`
- `data/`
- `profiles/`
- `*.db`、`*.db-wal`、`*.db-shm`
- 运行日志和导出文件
- 包含账号、Cookie、手机号、地址、订单号或支付信息的截图与配置

当前 `.gitignore` 已覆盖以上主要本地数据。正式执行 `git add` 后，仍应先运行 `git status`，确认暂存列表中没有账号资料和数据库，再进行提交与推送。
